White Gradient

On the portfolio homepage, the background video of the tropical sea fades to white as the user scrolls down. Here's how the effect was achieved:

The Problem

I wanted my background gradient to span across a couple nested semantic containers. (I needed to maintain proper HTML structure for accessibility and SEO purposes, but this created a design challenge.) The HTML is structured something like this:

<main>

  <h2>Portfolio</h2>

  <section>

    <h3>Professional Projects</h3>

  </section>

</main>

The Solution

Two separate gradients that look like one continuous gradient. The top one fades from an opacity of 0 to 50%. The bottom one fades from 50% opacity to 100%.

I wrapped the top portion inside a "white-gradient-1" div, and the bottom portion inside a "white-gradient-2" div.

Then I styled each one accordingly with CSS.

<main>

  <div id="white-gradient-1">

    <h2>Portfolio</h2>

  </div>

  <section>

    <div id="white-gradient-2">

      <h3>Professional Projects</h3>

    </div>

  </section>

</main>

White Gradient 1

#white-gradient-1 {
  background-image: linear-gradient(to bottom, rgba(255, 255, 255, 0) 0, rgba(255, 255, 255, .5) 100%);
  padding: 7em 0 3.5em 0;
  width: 100%;
}

White Gradient 2

CSS - Tablet

You'll notice this size uses a variety of media queries, some using min-width, and some using both min-width and max-width. That's because some of these styles are going to start applying to the desktop layout, too.

I change all the background colors back to white and text to black.

The blurred image is simply no longer passed into the background, so its container reverts to being transparent. Likewise, no image is passed into the #img-sm container anymore; instead, a larger one gets passed into #img-md, which now serves as the background.

From here on up, the project-title-box has a width of max-content. It also gets a negative margin-bottom to draw the main image up a bit behind it.

In turn, the main text box gets a negative margin-top to draw it up on top of the main image, as well.

For tablet only, the main image gets a white gradient at the very bottom, letting it blend into the background and the main text box.

The buttons shift layout, but this is done with Bootstrap rows and columns. And now that the two secondary buttons will be side-by-side from here on up, I've given them some custom spacing to keep them symmetrical.

CSS - Desktop

Now the main image is large enough to handle having the text box overlay it. Therefore, I give it a position: relative so that I can give the text box position: absolute.

From there, I can attach the text box to the bottom using bottom: 0, set a width of 40%, and let the top get as tall as it needs to based on its contents.

Then, depending on whether I want the text box to be on the left or right (I want to alternate with each project), I position it 3% from the edge.

Because the screen is now bigger, I pass larger versions of the background image into #img-lg and pass nothing into #img-sm or #img-md.

CSS - Mobile

Because I wrote my HTML mobile-first, the layout is already pretty straightforward for the small screens.

Now to position and style them:

I give each one a border and a subtle border radius.

I give each one's parent container display: inline-block to align them side-by-side.

I add hover effects, since each one is a clickable link.

Then I set the root height and width for the entire icon container to 100px. (I don't normally like to hardcode pixel values for heights or even widths, but in this case I wanted a perfect square, plus I'm going to use this root value to calculate several other things later on.) I chose 100px both for style and so that they'd be big enough for a finger to easily press on a touchscreen.)

I then let CSS calculate the icon font size to be half that of the root size I gave the container. This will pay dividends for all the rest of the screen sizes, as I will be able to adjust only the root size and everything will stay in ratio.

The gray boxes automatically stack vertically because of the Bootstrap grid I'm using: <div class="col-12 col-md-6 col-lg-5">.

CSS - Desktop

Now the fun begins, because this is the real layout I was inspired to do in the first place!

The icons are now switched to display: block, the margins are removed, and their position is changed to position: absolute.

The first one is then positioned all the way to the left, with a negative top margin equal to half its height.

The second one is then positioned all the way to the right, with a negative top margin equal to half its height.

The third one is then positioned all the way to the right, with a calculated top margin of 25% of the root width, plus the root width.

To position the header image behind all the icons and set its corners at their centers, I give its parent container equal margins of half the size of each icon, pushing it inward.

Although the gray Objective box is in a separate row beneath the image, I want it to overlap. This is easily achieved with a negative top margin and z-index: 2.

I also want the gray Process box to align with the right side of the header image, so I give that a margin-right of half the size of the icons — just like I did for the image itself.

And then, best of all, at screen sizes above 1200px, I simply adjust the icon size from 100px to 150px, and everything adjusts accordingly: the icon sizes, their borders, the margins of the header image, and the alignment of the gray process box. Life is good!

Responsive Font Sizes

Here's how I maintained the desired font size ratio for each heading. (Hint: I set the ratios once and only adjusted the root font size at each CSS media query, significantly reducing the amount of code I needed to write.)

Select Font Size Ratio

First, I used Type Scale to decide on a ratio. There was some trial and error involved here because there were many other variables to take into account. Font, font weight, and the length of the text content, which could vary from heading to heading and from page to page.

Suffice it to say I landed on Augmented Fourth, which uses a ratio of 1.414.

Write D.R.Y. Code

Now that I knew my ratio, I coded it into its own CSS file called "Global Fonts." Keeping my font code in its own separate file made for easier editing later.

It was also a convenient way to follow the D.R.Y. coding principle (Don't Repeat Yourself). I didn't want to rewrite these same ratios for each page — I wanted to write them once and point all my pages to this one file.

In any case, the real advantage of setting all the ratios up front is that when I want to adjust the font sizes for each breakpoint, I will only have to adjust the root font size, and everything else will resize automatically. (More on this later.)

/* FONT SIZE RATIOS */
#project-container h1 {
  font-size: 5.653em;
}

#project-container h2 {
  font-size: 3.998em;
}

#project-container h3 {
  font-size: 2.827em;
}

#project-container h4 {
  font-size: 1.999em;
}

#project-container h5 {
  font-size: 1.414em;
}

#project-container h6 {
  font-size: 1em;
}

Set Minimum Font Size

For readability, I didn't want my body copy to ever get smaller than 16px, so I hardcoded that in across all screen sizes.

#project-container p {
  font-size: 1rem;
}

Exception
